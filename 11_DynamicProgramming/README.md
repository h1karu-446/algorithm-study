# 動的計画法（Dynamic Programming）

問題を**小さな部分問題に分解し、その結果を再利用することで効率よく解く手法**。

## 基本アイデア

同じ計算を何度も行う問題に対して、

- 一度計算した結果を記録しておく
- 次に必要になったときは再計算せず、その値を使う

ことで計算量を削減する。

## フィボナッチ数列

フィボナッチ数列は以下で定義される：

- f(0) = 1
- f(1) = 1
- f(n) = f(n-1) + f(n-2)

例：
1, 1, 2, 3, 5, 8, ...


## 再帰による問題点

再帰でそのまま実装すると：

`fibonacci(n) = fibonacci(n-1) + fibonacci(n-2)`


同じ値を何度も計算してしまう。

### 例：f(5) の計算

- f(5) → f(4) + f(3)
- f(4) → f(3) + f(2)
- f(3) → f(2) + f(1)

このように、  
**f(3), f(2), f(1), f(0)** が何度も計算される。

→ 計算量は指数時間（非常に遅い）


## 動的計画法による解決

一度計算した値を配列に保存することで、

- 同じ計算を繰り返さない
- 計算量を大幅に削減できる

このような手法を**メモ化（memoization）**という。

特に、再帰を使ったメモ化をメモ化再帰という.
