# 動的計画法（Dynamic Programming）

問題を**小さな部分問題に分解し、その結果を再利用することで効率よく解く手法**。

## 基本アイデア

同じ計算を何度も行う問題に対して、

- 一度計算した結果を記録しておく
- 次に必要になったときは再計算せず、その値を使う

ことで計算量を削減する。

## フィボナッチ数列

フィボナッチ数列は以下で定義される：

- f(0) = 1
- f(1) = 1
- f(n) = f(n-1) + f(n-2)

例：
1, 1, 2, 3, 5, 8, ...


### 再帰による問題点

再帰でそのまま実装すると：

`fibonacci(n) = fibonacci(n-1) + fibonacci(n-2)`


同じ値を何度も計算してしまう。

### 例：f(5) の計算

- f(5) → f(4) + f(3)
- f(4) → f(3) + f(2)
- f(3) → f(2) + f(1)

このように、  
**f(3), f(2), f(1), f(0)** が何度も計算される。

→ 計算量は指数時間（非常に遅い）


### 動的計画法による解決

一度計算した値を配列に保存することで、

- 同じ計算を繰り返さない
- 計算量を大幅に削減できる

このような手法を**メモ化（memoization）**という。

特に、再帰を使ったメモ化をメモ化再帰という.

## 最長共通部分列 (Longest Common Subsequence Problem)

最長共通部分列 (LCS) とは、  
2つの列に共通して現れる**順序を保った部分列**（連続している必要はない）のうち、最も長いものを指す。

> 例  
> X = "ABCDE", Y = "ACED" のとき  
> 最長共通部分列 (LCS) は長さ3の "ACE"

m : X の長さ  
n : Y の長さ  

C[i][j] を「X の先頭 i 文字と Y の先頭 j 文字における LCS の長さ」とする。

このとき、動的計画法により以下の漸化式で求められる。

### 漸化式

```
C[i][j] 
= 0 (i = 0 または j = 0の時),  
= C[i-1][j-1] + 1 (X[i] = Y[j]の時)
= max(C[i-1][j], C[i][j-1]) (X[i] != Y[j]の時)
```

### ポイント

- 文字が一致する場合 → LCS を1つ伸ばす
- 一致しない場合 → どちらかを捨てた場合の最大値を取る
