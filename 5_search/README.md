# 線形探索 (Linear Search)

配列の先頭から順番に要素を調べ、目的の値を探すアルゴリズム。  
データがソートされていなくても使用できるのが特徴。  
計算量は `O(n)`。

## 番兵法 (Sentinel)

探索する配列の末尾に探索対象の値（番兵）を追加することで、  
境界チェックを省略し、比較を一致判定のみで行えるようにする手法。

# 二分探索 (Binary Search)

あらかじめ昇順または降順にソートされている配列に対して使用できる探索アルゴリズム。  
探索範囲の中央の要素と探索値を比較し、その大小関係に応じて探索範囲を半分に絞り込んでいく方法。

- 継続条件:`left < right`, 更新:`right = mid`
- 継続条件:`left <= right`, 更新:`right = mid-1`  
の2通りがある

# ハッシュ (Hash)

データを **ハッシュ関数**によって配列のインデックスに変換し、  
高速に探索・挿入を行うデータ構造。

通常の探索では `O(n)` かかるが、ハッシュを用いると  
平均 `O(1)` で探索できる。

## ハッシュ関数 (Hash Function)

キー（key）を **配列の添字（index）** に変換する関数。

例：`h(k) = k mod m` mは配列のサイズ Mは**素数**の方が分布が均一になりやすい


## 衝突 (Collision)

異なるキーが **同じハッシュ値**になること。

### チェイン法 (Chaining)

同じハッシュ値の要素を **リストでつなぐ方法**

### オープンアドレス法 (Open Addressing)

衝突した場合、**別の空き場所を探して格納する方法**
