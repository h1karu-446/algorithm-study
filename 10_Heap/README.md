# ヒープ

## 完全二分木

完全二分木とは、

最下段を除いてすべての段が埋まっており、最下段は左から順に詰めてノードが配置される二分木である。

満杯二分木 :「すべての葉が同じ深さ・内部ノードの子が2つ」*これを完全二分木ということもある。


### 配列による表現

完全二分木はノードが詰めて配置されるため、1オリジンの配列で表現できる。

ノードの添字を i とすると：

- 親：floor(i / 2) * floor(x)はx以下の最大の整数
- 左の子：2i
- 右の子：2i + 1


### 計算量の特徴

ノード数を n とすると、木の高さは log2(n) となるため、

- 挿入
- 削除

などの操作を O(log n) で行える。


## 二分ヒープ

完全二分木のうち、以下のヒープ条件を満たすもの。

- maxヒープ：親のキー >= 子のキー
- minヒープ：親のキー <= 子のキー

※兄弟ノード間には大小関係の制約はない

### ヒープの構築（maxヒープ）

配列からヒープを構築する場合、下から順に調整（heapify）していく。

1. 最後の内部ノード（floor(n / 2)）からルートまで以下を繰り返す
2. 自分と子ノードを比較
3. 大きい方の子と交換しながら下へ移動
4. 葉に到達するまで繰り返す

### 補足

- 内部ノードの範囲：
  1 〜 floor(n / 2)

- 葉ノードの範囲：
  floor(n / 2) + 1 〜 n

## 優先度付きキュー（Priority Queue）

値の大小などの「優先度」に基づいて、要素を取り出す順番が決まるデータ構造。

通常のキュー（FIFO：先入れ先出し）とは異なり、挿入順ではなく優先度によって取り出される。

### 実装
ヒープ（Heap）を用いて実現する。

- 最大値を優先する場合：maxヒープ
- 最小値を優先する場合：minヒープ

### 操作

- **挿入（insert）**  
  要素をヒープの末尾（最も右の葉）に追加し、親ノードと比較しながらヒープ条件を満たす位置まで上方向に交換していく

- **取り出し（extract）**  
  ルート（最大値または最小値）を取り出し、末尾の要素をルートに移動する    
  その後、子ノードと比較しながらヒープ条件を満たす位置まで下方向に交換していく
