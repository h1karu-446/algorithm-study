# ヒープ

## 完全二分木

完全二分木とは、

最下段を除いてすべての段が埋まっており、最下段は左から順に詰めてノードが配置される二分木である。

満杯二分木 :「すべての葉が同じ深さ・内部ノードの子が2つ」*これを完全二分木ということもある。


### 配列による表現

完全二分木はノードが詰めて配置されるため、1オリジンの配列で表現できる。

ノードの添字を i とすると：

- 親：floor(i / 2) * floor(x)はx以下の最大の整数
- 左の子：2i
- 右の子：2i + 1


### 計算量の特徴

ノード数を n とすると、木の高さは log2(n) となるため、

- 挿入
- 削除

などの操作を O(log n) で行える。


## 二分ヒープ

完全二分木のうち、以下のヒープ条件を満たすもの。

- maxヒープ：親のキー >= 子のキー
- minヒープ：親のキー <= 子のキー

※兄弟ノード間には大小関係の制約はない

## ヒープの構築（maxヒープ）

配列からヒープを構築する場合、下から順に調整（heapify）していく。

1. 最後の内部ノード（floor(n / 2)）から開始
2. 自分と子ノードを比較
3. 大きい方の子と交換しながら下へ移動
4. 葉に到達するまで繰り返す

## 補足

- 内部ノードの範囲：
  1 〜 floor(n / 2)

- 葉ノードの範囲：
  floor(n / 2) + 1 〜 n
