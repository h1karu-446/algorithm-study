# MergeSort

分割統治法（Divide and Conquer）に基づくソートアルゴリズムである。

配列を再帰的に2つに分割し、それぞれをソートした後、  
すでにソートされた2つの配列を **マージ（merge：合併）** することで  
全体をソートする。

マージ処理は「ソート済みの2つの配列を1つのソート済み配列に結合する」  
アルゴリズムを基礎としている。

計算量は **O(n log n)** である。

また、マージソートは **安定ソート** であり、  
補助配列として **O(n)** の追加メモリを必要とする。

# Partition（仕分け）

Partition はクイックソートで用いられる処理で、  
**基準値（pivot）をもとに配列を2つのグループに分けるアルゴリズム**である。

基準値以下の要素を左側に、基準値より大きい要素を右側に配置するように  
配列を並び替える。

典型的な実装では、配列の最後の要素を **pivot** とし、  
左から順に要素を確認しながら「pivot 以下の要素の領域」を  
徐々に広げていくことで並び替えを行う。

処理が終了すると、pivot は  
「左側が pivot 以下、右側が pivot より大きい」  
という位置に配置される。

- 計算量: **O(n)**
- 安定性: **不安定ソート**

# QuickSort

Quick Sort（クイックソート） は
Partition を利用した分割統治法のソートアルゴリズムである。

手順は次の通り。

1. 配列から pivot を選ぶ

2. partition によって配列を
pivot 以下
pivot より大きい
の2つの部分配列に分割する

3. それぞれの部分配列に対して 再帰的に quick sort を適用する

pivotの選び方によって再帰の回数が大きく変わるため、ランダムな値を選んだり、適当な値を複数選びその中央値を設定したり工夫が必要

- 平均計算量: O(n log n)
- 最悪計算量: O(n²)
- 不安定ソート
- 追加メモリが不要


# CountingSort
各値の出現回数を数えることでソートを行うアルゴリズム

比較を行わず、配列内の要素の値そのものをインデックスとして利用し、要素の個数を記録することで整列を行う。

値の最大値k 個の要素の配列が必要になるため、大きな値を持つ配列には適していない。

1. 出現回数を数える `C[x] = 値xの出現回数`
2. 累積和を計算する `C[x] = C[x] + C[x-1]` C[x-1]がすでに累積和なのでこれでok
3. 元の配列を後ろから走査し、要素の正しい位置に配置する。後ろから処理することで安定なソートになる。

- 時間計算量O(n+k)  nは要素数, kは値の最大値
- 空間計算量O(n+k)
- 安定ソート
- 整数データに用いられる
