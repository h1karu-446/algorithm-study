# 木構造
ノード(根)とエッジ(辺)で表されるデータ構造。階層構造に適している

## rotted tree
root(根)と呼ばれる他と区別された1つの特定の頂点を基準に階層構造を持たせた木。

- 親：あるノードの1つ上のノード
- 子：あるノードの1つ下のノード
- 次数：そのノードにおける子の数
- 兄弟：同じ親を持つノード
- 葉：子を持たないノード
- 内部ノード：葉以外のノード
- 深さ；rootからそのノードまでのedgeの数
- 高さ：そのノードから一番深い葉までの距離
- 部分木：あるノードをrootとする木
- 順序木：子に順序性がある木

rootは唯一親を持たない特別なノード

## 二分木
ルートを除いて全てのノードの次数が2以下である木。  
左の子か右の子かは区別する(順序木)。

二分木Tは、以下のいずれかの条件を満たす木と再帰的に定義できる。
- Tはノードを全く持たない(つまり空の木)
- または、Tは次の3つの集合から構成される  
  - 根(root)
  - 左部分木
  - 右部分木
  
この時左部分木と右部分木も**それぞれ二分木**である

例えば、
```
      A
     / \
    B   C
   / \
  D   E
この時、
root = A
左部分木 = B-D-E
右部分木 = C

そして、左部分木も再帰的に
root = B
左部分木 = D
右部分木 = E
となる
```

## 二分木の巡回
1. PreOrder (行きがけ順)  
`root → 左部分木 → 右部分木` の順にノードを辿る 

2. InOrder (通りがけ順)  
`左部分木 → root → 右部分木` の順にノードを辿る

3. PostOrder (帰りがけ順)  
`左部分木 → 右部分木 → root` の順にノードを辿る



## 木の実装

| 方法 | 特徴 | 簡単な実装設計（C++イメージ） |
|---|---|---|
| 親配列 (Parent Representation) | 各ノードが「親」を1つ持つ。実装が最も簡単。rootは -1 などで表す。子ノードを取得するには全体を走査する必要がある。 | ```int parent[N]; parent[i] = 親ノード ``` |
| 子リスト (Children List) | 各ノードが「子ノードのリスト」を持つ。子の列挙が簡単で木構造を自然に表せる。親を知るには別途記録が必要。 | ```vector<int> children[N]; children[i] = [1, 3, 4] ``` |
| 隣接リスト (Adjacency List) | グラフとして木を表現。競技プログラミングで最もよく使う。DFS/BFSと相性が良い。親子関係は探索中に決める。 |  |
| 左子右兄弟 (Left Child Right Sibling) | 任意木を二分木として表現できる。各ノードが「最左の子」と「右の兄弟」を持つ。AOJなどのアルゴリズム教材でよく出る。 | ```struct Node {int parent; int left; int right; }; Node tree[N];``` |
二分木(Binary Tree) | 各ノードに親と左の子と右の子を持たせる。| struct Node { int parent; int left; int right}
